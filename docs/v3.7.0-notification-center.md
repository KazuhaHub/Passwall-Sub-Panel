# v3.7.0 统一告警 / 通知中心 设计稿

把目前散在 dashboard 的多类告警（节点健康 / 证书 / 即将到期用户）收敛成**一个 typed Alert 抽象 + 单一 feed + 顶栏通知中心**，并新增「面板可升级（已测支持）」告警。与 [[v3.7.0-account-security]] 同属 v3.7.0、协同（锁定/异常登录是一类安全告警）。

> 中文文档；代码注释英文；单一真相源 + drift 防护（[[feedback_drift_guard]]）；TDD（[[feedback_tdd]]）。

## 0. 现状
[admin_dashboard.go](internal/transport/http/handler/admin_dashboard.go) 的 `Summary()` 已聚合三类（`node_alerts` / `cert_alerts` / `expiring_users`），前端 [DashboardView](web-react/src/views/admin/DashboardView.tsx) 用复用的 `AlertCard` 渲染。问题：每类各写各的 DTO/卡片，没有统一模型、没有顶栏入口、加新类型要改多处。

## 1. 核心决策：派生式（derived），不建事件库
告警**每次请求实时从当前状态派生**（节点健康、证书状态、面板版本、用户到期…），**不落 `alerts` 事件表**。条件消失，告警自动消失（无需"自动 resolve"生命周期）。每条告警有稳定 `key`（如 `node_health:12`、`panel_upgrade:5:v3.2.9`、`cert_failed:3`）。
- 好处：零生命周期管理、永远反映真实状态、实现简单。
- 代价：无历史、无"已读/未读"水位。v1 的角标 = **当前活跃告警数**(按最高 severity 上色)。
- **可选 v2**：小 `alert_dismissals(key, dismissed_at)` 表支持「忽略」，re-fire 时（key 变或条件复现）再现。先不做，按需加。

## 2. 后端
**统一模型**（service 层）：
```
Alert{ Key, Type, Severity(error|warning|info), Title, Message, Target, Link, Since }
```
**AlertService 聚合器**——按需从各来源拉，零新表：
| Type | severity | 来源 |
|---|---|---|
| `node_health` | error | Node/XUIPanel repo：HealthState ∈ {panel_unreachable, inbound_missing, inbound_disabled, unreachable} |
| `cert_failed` | error | cert repo：最近签发/续期失败 |
| `cert_expiring` | warning | cert repo：active 但临近到期未续/已过期 |
| `panel_upgrade` | info | 3X-UI 面板版本 < PSP **已测最高支持版本**（`version.XUIUpgradeTarget` = ActiveMaxTestedXUI；**beta.17 改**：原先看上游 `LatestXUI()`，现只在「升到 PSP 已验证支持的版本」时提示，不追上游 latest） |
| `psp_upgrade` | info | **PSP 自身**有新稳定版（GitHub `/releases/latest`，预发布感知比较 `pspBehindStable`，所以 beta 落后于其稳定版也会提示；**beta.17 新增**，admin-only，只提示稳定版） |
| ~~`user_expiring`~~ | — | **beta.17 移出通知中心**（通知应只与管理员相关）；用户即将到期改由 dashboard 卡片 + 邮件提醒承载，不再进铃铛 |
| `login_security`（随账号安全 beta 接入）| warning | auth_events / lockout：近期锁定 / 爆破峰值 |

**端点**：`GET /admin/alerts` → `{alerts:[...], counts:{error,warning,info}}`。**单一真相源**——dashboard 的卡片改为消费这个 feed（消除现在各写一份的 drift；加 drift 测试钉死"dashboard 用到的类型 ⊆ AlertService 产出"）。

## 3. 前端
- **顶栏铃铛 + 角标**（AdminLayout）：数字 = 活跃告警数，颜色 = 最高 severity；点击弹下拉列表（按 severity 分组、图标按 type、点条目深链跳对应页/操作，如 panel_upgrade → 升级）。
- **可选独立 `/admin/alerts` 页**：全量、可筛选（v1 可先只做铃铛下拉，列表页按需）。
- Dashboard 现有 `AlertCard` 区**改为消费统一 feed**（同源，不再各查各的）。

## 4. 分阶段（插进 v3.7.0 beta 序列）
建议顺序（与账号安全交错，让安全告警能接进来）：
1. account-security **beta.1** 验证码 + 锁定（✅ 已实现）
2. **通知中心 beta（= v3.7.0-beta.2，✅ 已实现）**：`internal/service/alert` 派生式 AlertService（node_health/cert_failed/cert_expiring/panel_upgrade/user_expiring/login_security 六类）+ `GET /admin/alerts`（按角色过滤 admin-only 类）+ 顶栏铃铛/角标/下拉 + `panel_upgrade`（零新 GitHub）+ `login_security`（`CountByReasonSince` 数 `locked_out`）+ drift 测试（窗口一致 + 类型 ⊆）。**dashboard 卡片暂仍各查各的**（保持工作），靠 drift 测试钉死不漂移；完全合并到同源 feed 留后续清理。
3. account-security 续：找回 → 注册 → 2FA → Passkey（各自 beta 里若产生安全事件，emit 对应 alert）
4. → v3.7.0 正式版汇总

## 5. 与现有的关系
派生式 = 不动数据模型、不加表（v1）。dashboard 改为同源消费（去重、防 drift）。`panel_upgrade` 只报**已测支持**的升级，不怂恿升进 untested。所有告警只读 + 深链，不在通知中心里直接执行危险操作（升级仍走带二次确认的升级流程）。

## 6. beta.17 调整（落地与设计差异）
- **`panel_upgrade` 改判据**：从「上游 `LatestXUI()` + 已测支持」改为「面板 < PSP 已测最高支持版本」（`version.XUIUpgradeTarget`）。语义更准：只在「升到 PSP 验证过的版本」时提示，不再受上游发了多新影响。
- **`user_expiring` 移出通知中心**：通知中心定位为「只与管理员相关」。用户即将到期是面向**用户**的事，保留 dashboard 卡片 + 邮件提醒，但不再进铃铛。
- **新增 `psp_upgrade`（PSP 自更新检测）**：读 GitHub `/releases/latest`（按 `prerelease` + tag 串双重过滤，只认稳定版），`pspBehindStable` 预发布感知比较——跑 beta 而其稳定版已发时也提示。admin-only。
- **渠道色**：顶栏版本号按通道上色（绿 = 稳定、黄 = 预发布、灰 = dev）。与 docker `:latest`/`:beta` 两条通道、release.yml 的 `prerelease` 标记同一套「稳定 vs 前沿」理念。
- 真相源仍是代码（[alert.go](internal/service/alert/alert.go) / [latest_psp.go](internal/version/latest_psp.go) / [compat.go](internal/version/compat.go)）；本稿为设计记录，v3.7.0 正式版后按惯例可删。
